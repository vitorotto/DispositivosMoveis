Tutorial Abrangente de Classes em Dart (Null-Safe)

Este tutorial explora classes em Dart (versão 3.5) com suporte a null safety. Ele cobre definição de classes, construtores, herança, mixins, getters/setters, sobrecarga de operadores, classes abstratas, interfaces e membros estáticos. Todos os exemplos são comentados em português para facilitar o aprendizado de alunos.

1. Definindo uma Classe

Classes em Dart são usadas para criar objetos com propriedades e métodos. Com null safety, todas as propriedades devem ser inicializadas ou marcadas como nullable (`?`) ou `late`.

Exemplo 1: Classe Simples


class Estudante {
  // Propriedades não-nuláveis (devem ser inicializadas)
  final String nome;
  final int idade;
  final double nota;

  // Construtor com parâmetros requeridos
  Estudante({required this.nome, required this.idade, required this.nota});

  // Método para verificar aprovação
  bool estaAprovado() {
    return nota >= 6.0;
  }

  // Método para exibir informações
  void exibirInfo() {
    print('Estudante: $nome, Idade: $idade, Nota: $nota, Aprovado: ${estaAprovado() ? "Sim" : "Não"}');
  }
}

// Função principal para testar
void main() {
  // Criando um objeto com parâmetros nomeados
  var estudante = Estudante(nome: 'João', idade: 20, nota: 7.5);
  estudante.exibirInfo(); // Saída: Estudante: João, Idade: 20, Nota: 7.5, Aprovado: Sim
}

Explicação: A classe Estudante usa propriedades final (imutáveis) e um construtor com parâmetros required para garantir inicialização. Null safety exige que todas as propriedades sejam inicializadas no momento da criação do objeto.

Exemplo 2: Classe com Lista Nullable


class Curso {
  // Propriedades
  final String nome;
  List alunos; // Lista não-nulável, inicializada no construtor

  // Construtor com lista inicializada como vazia por padrão
  Curso(this.nome, {List? alunos}) : alunos = alunos ?? [];

  // Método para adicionar aluno
  void adicionarAluno(String aluno) {
    alunos.add(aluno);
    print('$aluno foi adicionado ao curso $nome');
  }

  // Método para exibir alunos
  void exibirAlunos() {
    print('Curso: $nome');
    print('Alunos: ${alunos.isEmpty ? "Nenhum aluno" : alunos.join(", ")}');
  }
}

// Função principal para testar
void main() {
  // Criando um curso com lista vazia
  var curso = Curso('Programação em Dart');
  curso.adicionarAluno('Maria'); // Saída: Maria foi adicionado ao curso Programação em Dart
  curso.adicionarAluno('Pedro');
  curso.exibirAlunos(); // Saída: Curso: Programação em Dart, Alunos: Maria, Pedro
}

Explicação: A propriedade alunos é inicializada no construtor com uma lista vazia se nula (alunos ?? []). O parâmetro alunos é nullable (List?) para permitir flexibilidade na criação.

2. Construtores

Dart suporta construtores padrão, nomeados e factory, todos adaptados para null safety.

Exemplo 1: Construtor Padrão e Nomeado


class Produto {
  // Propriedades
  final String nome;
  double preco;

  // Construtor padrão
  Produto({required this.nome, required this.preco});

  // Construtor nomeado para promoção
  Produto.emPromocao({required this.nome, required double precoOriginal})
      : preco = precoOriginal * 0.8;

  // Método para exibir detalhes
  void exibirDetalhes() {
    print('Produto: $nome, Preço: R\$${preco.toStringAsFixed(2)}');
  }
}

// Função principal para testar
void main() {
  var produto1 = Produto(nome: 'Celular', preco: 1000.0);
  var produto2 = Produto.emPromocao(nome: 'Notebook', precoOriginal: 2000.0);
  produto1.exibirDetalhes(); // Saída: Produto: Celular, Preço: R$1000.00
  produto2.exibirDetalhes(); // Saída: Produto: Notebook, Preço: R$1600.00
}

Explicação: O construtor usa required para parâmetros obrigatórios. O construtor nomeado emPromocao calcula o preço com desconto. Todas as propriedades são inicializadas para atender ao null safety.

Exemplo 2: Construtor Factory


class Singleton {
  // Propriedade estática para a única instância
  static final Singleton _instancia = Singleton._interno();

  // Propriedade
  final String nome;

  // Construtor privado
  Singleton._interno() : nome = 'Instância Única';

  // Construtor factory para retornar a mesma instância
  factory Singleton() => _instancia;

  // Método para exibir mensagem
  void exibirMensagem() {
    print('Esta é a $nome');
  }
}

// Função principal para testar
void main() {
  var s1 = Singleton();
  var s2 = Singleton();
  s1.exibirMensagem(); // Saída: Esta é a Instância Única
  print(identical(s1, s2)); // Saída: true
}

Explicação: O padrão Singleton usa um construtor factory para retornar a mesma instância. A propriedade nome é final e inicializada diretamente, garantindo null safety.

3. Herança

Herança permite reutilizar código, com null safety garantindo inicialização adequada.

Exemplo 1: Herança Simples


class Veiculo {
  // Propriedades
  final String marca;
  final int ano;

  // Construtor
  Veiculo({required this.marca, required this.ano});

  // Método que pode ser sobrescrito
  void mover() {
    print('$marca está se movendo');
  }
}

class Carro extends Veiculo {
  // Construtor que inicializa a classe pai
  Carro({required String marca, required int ano}) : super(marca: marca, ano: ano);

  // Sobrescrevendo o método
  @override
  void mover() {
    print('Carro $marca do ano $ano está acelerando na estrada');
  }
}

// Função principal para testar
void main() {
  var carro = Carro(marca: 'Toyota', ano: 2022);
  carro.mover(); // Saída: Carro Toyota do ano 2022 está acelerando na estrada
}

Explicação: A classe Carro herda de Veiculo usando extends. Propriedades são final e inicializadas via construtor com required. O método mover é sobrescrito.

Exemplo 2: Mixins


mixin Eletrico {
  // Método do mixin
  void carregarBateria() {
    print('Carregando bateria elétrica');
  }
}

class CarroEletrico extends Veiculo with Eletrico {
  // Construtor
  CarroEletrico({required String marca, required int ano}) : super(marca: marca, ano: ano);

  @override
  void mover() {
    print('Carro elétrico $marca do ano $ano está se movendo silenciosamente');
  }
}

// Função principal para testar
void main() {
  var carro = CarroEletrico(marca: 'Tesla', ano: 2023);
  carro.mover(); // Saída: Carro elétrico Tesla do ano 2023 está se movendo silenciosamente
  carro.carregarBateria(); // Saída: Carregando bateria elétrica
}

Explicação: O mixin Eletrico adiciona funcionalidade à CarroEletrico. Null safety é garantido pela inicialização das propriedades no construtor da classe pai.
